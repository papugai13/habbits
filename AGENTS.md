# AGENTS.md — Инструкции для ИИ-агентов (Habits Project)

Этот файл содержит основные принципы, стандарты и контекст для работы над проектом **Habits** — приложением для отслеживания привычек.

## 🚀 Обзор проекта
**Habits** — это современное веб-приложение для трекинга привычек с аналитикой, ачивками и поддержкой категорий.

### Стек технологий:
- **Frontend**: React 19, Recharts (графики), Vanilla CSS.
- **Backend**: Django 5.2, Django REST Framework (DRF), MySQL/SQLite.
- **Deployment**: Nginx, Gunicorn, systemd.

---

## 🏗 Архитектура и структура
- `/backend/`: Django проект. Основная логика в приложении `api`. Обрати внимание, что `views.py` содержит много логики.
- `/frontend/`: React приложение. Большинство компонентов и логики сосредоточено в `App.js` и директории `components`.
- `/nginx.conf`: Конфигурация для продакшена.
- `requirements.txt`: Зависимости бэкенда.

### 🧩 Модульность и структура
- **Избегай гигантских файлов**: Если файл (например, компонент React или View в Django) становится слишком большим, старайся выносить логику в отдельные файлы, хуки или вспомогательные функции.
- **Чистота кода**: Код должен быть разбит на логические блоки. Это упрощает навигацию и поддержку.
- **Рефакторинг**: При работе с `App.js` или `views.py`, если видишь возможность вынести часть кода в отдельный компонент или сервис без риска поломки — делай это.

## 📜 Основные правила разработки

### 🎨 Frontend и Дизайн
- **Эстетика прежде всего**: Дизайн должен выглядеть премиально и современно (стекломорфизм, плавные градиенты, микро-анимации).
- **CSS**: Используй чистый CSS (Vanilla CSS). Избегай Tailwind, если не поступило прямого запроса.
- **Интерактивность**: Используй Recharts для визуализации прогресса.
- **Адаптивность**: Приложение должно идеально работать как на десктопе, так и на мобильных устройствах.

### ⚙️ Backend (Django)
- **Модели**: Основная модель пользователя — `UserAll`. Привычки связаны через `Habit` и `Date`.
- **Локализация**: Используй русский язык для `verbose_name` в моделях и для всех строк, отображаемых пользователю.
- **API**: Все эндпоинты должны находиться в `api/`. Используй DRF Serializers и ViewSets.
- **Миграции**: Будь осторожен с миграциями в продакшене. Всегда проверяй целостность данных перед удалением полей.

### 📝 Языковая политика
- **Код**: Переменные, функции, классы — на английском языке.
- **Интерфейс и данные**: Русский язык (RU).
- **Документация**: Комментарии к коду и README могут быть на русском языке.

### 🛡 Качество и самопроверка
- **Тщательная проверка**: Всегда перепроверяй код на наличие логических ошибок перед сохранением.
- **Целостность**: Большие файлы (`App.js`, `views.py`) требуют особого внимания. Изменение в одной части может сломать функционал в другой. Используй поиск по файлам для проверки зависимостей.
- **Тестирование**: Если возможно, проверяй UI изменения в браузере.
- **Безопасность**: Никогда не удаляй данные без явного подтверждения. Будь осторожен с `Overwrite: true`.
- **Следование паттернам**: Изучай существующий код перед внесением правок. Пиши в том же стиле, который уже используется в проекте.
